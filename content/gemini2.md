Do NOT summarize. Every line of dialogue, physical analogy, and board update must be accounted for.

1. **Raw Audio Primary Extraction:** Prioritize the raw audio track; correct jargon using visual evidence.
2. **Refined First-Person Register:** Professional, grammatically sound English using (I, We).
3. **The `(i.e., ...)` Calibration Anchor:** Inject explicit inline LaTeX annotations for referenced formulas or skipped algebraic steps.

5. **Structural Rigor:** Use `\section{}`, `\subsection{}`, and standard math environments (`theorem`, `proof`, etc.).
6. **Visual Math Syncing:** Variables spoken while being written must be perfectly formatted in the corresponding `math_stroke`.
7. **No Data Loss:** Every board note must be logged.
8. **Eradicate "Naked Math":** ALL equations/diagrams must be wrapped in `math_stroke` or `orangeformula`.
9. **Fallback for the Illegible:** Use `\textbf{[Illegible formula]}` for non-dictated, unreadable board states.

## Execution Workflow
0. **Pre-Flight Check:** Confirm access to files and perform an internal 1-minute calibration.
1. **Analyze:** Extract audio and OCR simultaneously.
2. **Buffer:** Build logic in 1-minute sequential blocks.
3. **Render:** Generate final LaTeX code using custom semantic environments. Output ONLY raw LaTeX.
4. **The Continuation Protocol (Token Management):** Do not attempt to generate 20 minutes of high-density transcription in one block. To prevent token-limit degradation (approx. 3,000–4,000 tokens per output), you MUST restrict each response to **5–7 minute intervals**. Find a natural mathematical stopping point (e.g., end of a `\end{proof}` or `\end{math_stroke}`) near the 7-minute mark. Close the code block and output EXACTLY: `**[SYSTEM] Segment complete. Please prompt "Continue" for the remainder of the segment.**`
