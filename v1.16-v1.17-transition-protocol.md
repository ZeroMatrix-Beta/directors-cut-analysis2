# V1.16 to V1.17 Transition Protocol

This document outlines the architectural upgrades, behavioral patches, and prompt engineering strategies required to transition the Master Protocol (`gemini.md`) from version V1.16 to V1.17.

## 1. State Management: The "Snapshot vs. Delta" Paradigm
**Issue:** Cross-Segment State Collapse. When spanning the 9-11 minute boundary, the AI assumes the user remembers the previous board state and outputs only the "delta" (e.g., omitting items 1 and 2 of a list and only generating items 3 and 4).
**V1.17 Fix:** Introduce an explicit rule mandating that every `math-stroke` is a visually self-contained **snapshot**. If a list or derivation spans a boundary, the entire block must be completely restated.

**Exact Prompt Diff (Word-by-Word):**
*Before (V1.16):*
> **Cognitive Redundancy & Environment Separation (Cognitive Anchoring):** Each semantic environment must serve exactly one role, but mathematical concepts MUST be actively duplicated across them. **You MUST explicitly restate and replicate every formula, geometric constraint, or logical explanation** inside a `math-stroke`, `tikzpicture` node, or `explanation-of-steps` block exactly as written on the board, even if it was just dictated verbally in the preceding `spoken-clean` block. Do not omit board content just because it is already in the spoken text. This intentional redundancy acts as a **self-attention anchor**.

*After (V1.17):*
> **Cognitive Redundancy & Environment Separation (Cognitive Anchoring):** Each semantic environment must serve exactly one role, but mathematical concepts MUST be actively duplicated across them. **You MUST explicitly restate and replicate every formula, geometric constraint, or logical explanation** inside a `math-stroke`, `tikzpicture` node, or `explanation-of-steps` block exactly as written on the board, even if it was just dictated verbally in the preceding `spoken-clean` block. Do not omit board content just because it is already in the spoken text. **CRITICAL (Snapshot vs. Delta): Treat every `math-stroke` block as a completely self-contained visual snapshot of the board. NEVER treat a new segment as a "delta" that only outputs new information. If a list, derivation, or diagram spans a continuation boundary, you MUST completely restate all previously written items/equations in the new segment's `math-stroke` to maintain structural anchoring.** This intentional redundancy acts as a **self-attention anchor**.

## 2. Anti-Lazy Optimization: Strict `\setcounter` Enforcement
**Issue:** The AI relies on standard LaTeX `\begin{enumerate}` auto-numbering and skips the required `\setcounter` syncing, treating it as a conditional suggestion rather than a structural absolute.
**V1.17 Fix:** Reword the LaTeX Formatting section to make `\setcounter` mandatory for *every single* `\item` in a numbered list, explicitly forbidding reliance on auto-numbering. Simultaneously, explicitly authorize the `description` environment for named property lists to resolve the AI's internal stylistic conflict.

**Exact Prompt Diff (Word-by-Word):**
*Before (V1.16):*
> **For any lists, bullet points, or sequential steps, you MUST explicitly use `\begin{itemize}` or `\begin{enumerate}` environments; NEVER format lists manually using plain text numbers or dashes. If the lecturer draws bullets on the board, but verbally numbers them, use `\begin{enumerate}`. To explicitly sync the written numbers with the spoken numbers (especially if the lecturer skips a number), manually set the counter before the specific `\item` using the `\setcounter{enumi}{<value>}` command (where `<value>` is the desired number minus one, e.g., `\setcounter{enumi}{3} \item` outputs "4.").**

*After (V1.17):*
> **For any lists, bullet points, or sequential steps, you MUST explicitly use `\begin{itemize}`, `\begin{enumerate}`, or `\begin{description}` environments; NEVER format lists manually using plain text numbers or dashes. If the lecturer lists named properties (e.g., "Property 1: Additivity"), you are highly encouraged to use the `description` environment (e.g., `\item[Property 1:]`) to elevate the textbook flow. If the lecturer writes a strictly sequential numbered list, use `\begin{enumerate}`. CRITICAL: To explicitly sync the written numbers with the spoken numbers, you MUST manually set the counter before EVERY SINGLE `\item` using the `\setcounter{enumi}{<value>}` command (where `<value>` is the desired number minus one, e.g., `\setcounter{enumi}{3} \item` outputs "4."). This is an absolute requirement for 1-to-1 blackboard synchronization, regardless of whether the lecturer follows a standard sequence or skips numbers.**

## 3. Thinking-Token Minimization: Invisible Scratchpads
**Issue:** Autoregressive LLMs suffer from "tunnel vision" and poor spatial/logical planning during complex LaTeX generation.
**V1.17 Fix:** Introduce internal reasoning environments (which will be stripped by the C# backend before compilation):
*   `\begin{ai-tikz-planner-invisible-content}`: Forces painter's algorithm depth sorting before TikZ generation.
*   `\begin{ai-proof-skeleton-invisible-content}`: Forces high-level structural outlining before complex derivations.
*   `\begin{ai-type-check-invisible-content}`: Forces variable dimensionality declarations.
*   `\begin{ai-example-invisible-content}`: Forces concrete instantiation of abstract concepts.

## 4. The `(Recall: ...)` Logical Anchor
**Issue:** Vague verbal references to past theorems cause logical hallucinations in the subsequent mathematical blocks.
**V1.17 Fix:** Expand the existing `(i.e., ...)` variable calibration to include `(Recall: [Theorem/Concept])` in `spoken-clean` blocks, forcing the AI to load the correct mathematical constraints into active memory before generating derivations.

## 5. Color Refactoring: `dvipsnames`
**Issue:** The custom `prof[color]` palette restricts the AI from using its native training data for aesthetic choices and reduces LaTeX portability.
**V1.17 Fix:** Deprecate `profblue`, `profred`, etc., and instruct the AI to use standard `xcolor` `dvipsnames` (e.g., `MidnightBlue`, `BurntOrange`, `ForestGreen`).

## 6. Verbatim Pacing & Punctuation
**Issue:** Strict verbatim text is often unreadable without rigid punctuation rules to simulate the speaker's cadence.
**V1.17 Fix:** Explicitly enforce **em dashes** (` — `) for self-corrections/restarts and **ellipses** (`...`) for thinking time and pauses.

## 7. Shorthand Expansion (Human Override)
**Issue:** The AI naturally expands chalkboard shorthand (e.g., "Eucl isometry" -> "Euclidean isometry"). While earlier audits flagged this as a "fidelity failure," human review dictates this is actually desirable for readability.
**V1.17 Fix:** Explicitly encourage the AI to expand obvious chalkboard abbreviations into proper academic prose inside the `math-stroke` environment, preserving the exact shorthand only in the `spoken-clean` visual anchors.
